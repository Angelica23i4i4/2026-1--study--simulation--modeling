# Цель работы

Целью данной лабораторной работы является настройка рабочего окружения для работы с системой контроля версий Git, создание репозитория курса на основе шаблона, настройка SSH- и GPG-ключей, а также освоение работы с git flow.

# Задание

1. Установить необходимое программное обеспечение (Quarto, Node.js, git-flow).
2. Создать репозиторий курса на платформе GitVerse на основе шаблона.
3. Настроить SSH-ключ и привязать его к учётной записи GitVerse.
4. Создать и привязать GPG-ключ для подписи коммитов.
5. Настроить git для подписи коммитов.
6. Создать структуру курса и выполнить первый коммит.
7. Настроить git flow и выполнить релиз.

# Теоретическое введение

Git — распределённая система контроля версий, позволяющая отслеживать изменения в файлах и координировать работу нескольких разработчиков. Git flow — методология ветвления в Git, которая определяет строгую модель ветвления, предназначенную для управления релизами проекта. SSH-ключи используются для безопасной аутентификации при работе с удалёнными репозиториями, а GPG-ключи — для подписи коммитов с целью подтверждения их подлинности.

GitVerse — российская платформа для хостинга Git-репозиториев, аналогичная GitHub и GitLab.

## Настройка git flow

Была выполнена инициализация git flow командой `git flow init` с параметрами по умолчанию: ветка `master` для production-релизов, ветка `develop` для разработки, префиксы `feature/`, `bugfix/`, `release/`, `hotfix/`, `support/` для соответствующих типов веток и префикс `v` для тегов версий. После инициализации в репозитории появились ветки `develop` и `master`, при этом активной стала ветка `develop` (рис. 10).

![Инициализация git flow](images/10.png){#fig-010 width=70%}

## Создание релиза

Все ветки были отправлены на удалённый сервер командой `git push -u --all`. Затем был создан релиз версии 1.0.0 командой `git flow release start 1.0.0`, после чего была создана ветка `release/1.0.0` на основе `develop`. С помощью `standard-changelog --first-release` был сгенерирован файл `CHANGELOG.md` (рис. 11).

![Создание релиза и changelog](image/11.png){#fig-011 width=70%}

Файл `CHANGELOG.md` был добавлен в коммит с сообщением `chore(site): add changelog`. После этого релиз был завершён командой `git flow release finish 1.0.0`. В результате ветка `release/1.0.0` была слита в `master`, создан тег `v1.0.0`, выполнено обратное слияние в `develop`, а ветка релиза удалена (рис. 12).

![Завершение релиза](image/12.png){#fig-012 width=70%}
